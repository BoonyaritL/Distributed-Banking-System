# บทพูดนำเสนอโปรเจค Distributed Banking System

## 1. บทนำ (Introduction)
"สวัสดีครับ/ค่ะ วันนี้ผมจะมานำเสนอโปรเจค **Distributed Banking System** หรือระบบธนาคารจำลองแบบกระจายศูนย์ครับ
โปรเจคนี้พัฒนาด้วยภาษา **Java** โดยหัวใจสำคัญคือการใช้เทคโนโลยี **Java RMI (Remote Method Invocation)** ในการจัดการสื่อสารระหว่างเครื่อง Client และ Server ครับ"

---

## 2. คอนเซปต์หลัก (Core Concept)
"สิ่งที่โปรเจคนี้ต้องการนำเสนอคือการทำงานแบบ **Client-Server Architecture** ที่สมบูรณ์แบบครับ
- **ฝั่ง Server (Bank):** เปรียบเสมือนสำนักงานใหญ่ธนาคาร ทำหน้าที่เก็บข้อมูลลูกค้าและประมวลผลธุรกรรมทั้งหมด
- **ฝั่ง Client (ATM):** เปรียบเสมือนตู้ ATM ที่กระจายอยู่ทั่วไป ทำหน้าที่รับคำสั่งจากผู้ใช้ แล้วส่งไปประมวลผลที่ Server
- โดยที่ทั้งสองส่วนนี้ **ไม่จำเป็นต้องอยู่ที่เดียวกัน** สามารถรันคนละเครื่อง คนละสถานที่ได้ โดยคุยกันผ่านระบบ Network แบบ **Location Transparency** ครับ คือผู้ใช้รู้สึกเหมือนโปรแกรมทำงานในเครื่องเดียว แต่จริงๆ แล้วทำงานข้ามเครื่องครับ"

---

## 3. สถาปัตยกรรมระบบ (System Architecture)
"โครงสร้างของระบบประกอบด้วย 3 ส่วนหลักครับ:

1. **Bank Interface:**
   - เป็นตัวกลางหรือ 'ข้อตกลง' (Contract) ที่ระบุว่าธนาคารนี้ทำอะไรได้บ้าง เช่น `login`, `deposit`, `withdraw`
   - ทั้ง Client และ Server ต้องรู้จัก Interface ตัวเดียวกันนี้ถึงจะคุยกันรู้เรื่องครับ

2. **Bank Server (Implementation):**
   - เป็นส่วนที่ทำงานจริง (Implementation)
   - เมื่อรันขึ้นมา มันจะสร้าง **RMI Registry** (สมุดรายชื่อ) และลงทะเบียนตัวเองชื่อว่า 'Bank' ไว้
   - ข้อมูลลูกค้าทั้งหมดถูกเก็บแบบ Persistent Data ลงไฟล์ `data.txt` ทำให้ข้อมูลไม่หายแม้ปิด Server

3. **ATM Client:**
   - เป็นส่วนติดต่อผู้ใช้ (User Interface) ซึ่งผมได้พัฒนาทั้งแบบ **Command Line** และ **GUI**
   - เมื่อเริ่มทำงาน Client จะทำการ **Lookup** (ค้นหา) ชื่อ 'Bank' จาก Registry
   - เมื่อเจอแล้ว จะได้รับ **Stub** (ตัวแทน) กลับมา และเรียกใช้เมธอดผ่าน Stub นี้เสมือนเรียก method ในเครื่องตัวเองครับ"

---

## 4. การทำงาน (Workflow Demo)
*(สาธิตการรันโปรแกรมจริง)*
"ขออนุญาตสาธิตการทำงานจริงครับ:
1. เริ่มจาก **รัน Server** -> เพื่อโหลดข้อมูลลูกค้าและเปิดรับการเชื่อมต่อ
2. **รัน Client (GUI)** -> จำลองเป็นผู้ใช้งานตู้ ATM
3. ทำการ **Login** -> ส่ง username/password ไปเช็คที่ Server
4. ทดลอง **ฝาก/ถอนเงิน** -> ยอดเงินจะถูกอัปเดตทันทีแบบ Real-time และบันทึกลงไฟล์ที่ฝั่ง Server ครับ"
